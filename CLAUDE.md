# SixTONESクイズ アプリ仕様書

## プロジェクト概要
SixTONESファン向けのクイズWebアプリ。フリープレイと絆クイズの2モードを持ち、ポイント・ストーン（💎）システム、ランキング機能を備える。

## 技術スタック
- **フロントエンド**: Vanilla JS（ES Modules）、HTML、CSS
- **バックエンド**: Firebase Firestore（Realtime DBから移行済み）
- **ホスティング**: Firebase Hosting
- **ファイル構成**: 単一ファイル構成を維持（app.jsを分割しない）

  │    └─ 履歴詳細画面（historyDetailScreen）
  ├─ プロフィール画面（profileScreen）
  ├─ 💎獲得画面（stoneGetScreen）　※後々実装予定

- `showScreen('screenId')` 関数で統一

## ポイント（P）システム

### 獲得条件（フリープレイ正解時）
| 難易度 | 獲得P |
|--------|-------|
| 初級   | 1P    |
| 中級   | 2P    |
| 上級   | 3P    |

### ランク制（累計ポイントで昇格・降格なし）
| ランク  | 必要P（累計） |
|---------|--------------|
| 新規リスナー | 0〜          |
| 見習いンズ | 50〜         |
| チムスト | 150〜        |
| サタスペリスナー | 350〜        |
| おやーンズ | 700〜        |
| 最髙地優吾 | 1500〜       |

- ランクはランキング画面・プロフィール画面に表示
- 称号システム（現行）とは別軸

## ストーン（💎）システム

### 獲得条件
| 条件 | 獲得数 |
|------|--------|
| ログインボーナス（1日1回） | 💎1個 |
| 動画広告視聴（Androidアプリ実装時） | 💎2個 |

### 消費条件
| 用途 | 消費数 |
|------|--------|
| 絆クイズ 4〜1級 | 💎1個 |
| 絆クイズ 特級 | 💎3個 |
| 履歴件数 +5件拡張 | 💎5個 | ※上限100件

---

## フリープレイ仕様

- タイマー機能：**なし**
- 問題数：10問/セット
- 難易度選択：初級 / 中級 / 上級 / ランダム（全難易度混合）
- 解説ボタン：回答後に表示（ボタン押下でモーダル表示）
- ポイント：正解時に難易度別Pを加算

---

## 絆クイズ仕様

| 級   | 問題数 | 合格ライン | 💎消費 | 出題難易度       |
|------|--------|-----------|--------|-----------------|
| 4級  | 3問    | 1問正解   | 1個    | 初級のみ         |
| 3級  | 5問    | 2問正解   | 1個    | 初級＋中級       |
| 2級  | 7問    | 3問正解   | 1個    | 全難易度         |
| 1級  | 7問    | 5問正解   | 1個    | 中級＋上級       |
| 特級 | 7問    | 全問正解  | 3個    | 上級のみ         |

- タイマー：**15秒/問**
- 解説ボタン：回答後に表示
- 合格時：Xシェアボタンを表示
- 💎不足時：挑戦不可（不足メッセージを表示）

## クイズデータ構造

```js
{
  question: "問題文",
  options: ["選択肢A", "選択肢B", "選択肢C", "選択肢D"],
  correct: 0,           // 正解のインデックス（0始まり）
  difficulty: "初級",   // "初級" | "中級" | "上級"
  explanation: "解説テキスト",  // 空文字可（後から追加）
  randomizeOptions: true  // falseの場合は選択肢をシャッフルしない
}
```

## Firestore データ構造

### users/{username}
```
{
  username: string,
  points: number,           // 累計ポイント（リセットなし）
  stones: number,           // 現在のストーン数
  rank: string,            // "新規リスナー" | "見習いンズ" | ... | "最髙地優吾"
  correctCount: number,     // 累計正解数
  totalCount: number,       // 累計回答数
  historyLimit: number,     // 履歴表示件数（初期値: 5、最大: 100）
  loginBonusDate: string,   // 最終ログインボーナス取得日（YYYY-MM-DD）
  title: object,            // 称号オブジェクト（既存システム）
  seasonPoints: number,     // 今シーズンのポイント（月次リセット）
  grade: string,            // 絆クイズの最高取得級（"-" | "4級" | ... | "特級"）
  updatedAt: timestamp
}
```

### rankings/{seasonId}
```
// seasonId形式: "2025-03"（年-月）
{
  seasonNumber: number,     // 第Nシーズン
  seasonId: string,         // "2025-03"
  endsAt: string,           // シーズン終了日
  data: [                   // 上位50位まで保存
    {
      rank: number,
      username: string,
      points: number,        // そのシーズンのポイント
      rank: string,
      grade: string
    }
  ],
  // 過去シーズンは上位20位まで保持（11シーズン分）
}
```

### history/{username}/items/{autoId}
```
{
  question: string,
  options: array,
  correct: number,
  userAnswer: number,       // ユーザーが選んだインデックス（タイムアウト: -1）
  isCorrect: boolean,
  points: number,           // 獲得P（不正解・絆クイズは0）
  difficulty: string,
  explanation: string,
  mode: string,             // "free" | "bond"
  grade: string,            // 級（"4級"など）、フリープレイは""
  timestamp: timestamp
}
```

## ランキング仕様

- **シーズン**: 毎月リセット（月末23:59終了）
- **シーズン名**: 第Nシーズン（N=1, 2, 3...）
- **現在のランキング**: 上位50位表示
- **過去データ**: 上位20位を最大11シーズン分保持
- **ランキング集計**: 今シーズンのポイント（seasonPoints）で順位付け
- **表示項目**: 順位・ユーザー名・ポイント・ランク・絆最高級

---

## 履歴機能

- **初期表示件数**: 5件（新しい順）
- **最大件数**: 100件
- **拡張**: 💎5個消費で+5件
- **一覧表示**: 問題文・正解/不正解・答え・獲得P
- **詳細表示**: 問題文・解答・解説・獲得P・モード・日時

## 注意事項・実装ルール

1. **app.jsは単一ファイルを維持**（分割しない）
2. **firebase-config.jsはGitにコミットしない**（.gitignoreに追加済み）
3. **メダルという単語は使わない** → ストーン（💎）に統一
4. **タイマーはフリープレイでは使わない**（絆クイズのみ15秒）
5. 問題の `explanation` が空文字の場合は解説ボタンを非表示にする
6. ストーン不足時は絆クイズ・ツールの該当ボタンをグレーアウト＋メッセージ表示
